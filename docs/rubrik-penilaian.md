# Rubrik Penilaian Proyek

| No | Category | Criteria Description | Score Range | Weight |
|---|---|---|---|---|
| 1 | Project Setup & Structure | Project runs without errors; correct folder structure; dependencies managed properly; works on emulator/device. | 0-10 | 10% |
| 2 | UI & Design Implementation | Clean and consistent layout; proper use of components (View, Text, Image, Button); responsive design with Flexbox; visually appealing UI. | 0-20 | 20% |
| 3 | Functionality & Interactivity | Core app features function correctly; user interactions (TextInput, TouchableOpacity, etc.) work properly; no crashes; dynamic updates work. | 0-20 | 20% |
| 4 | State Management & Hooks | Proper use of useState, useEffect, or other hooks; logical data flow; reusable components; correct UI reactivity to state. | 0-15 | 15% |
| 5 | Navigation & Data Flow | Correct setup of navigation (Stack, Tab, Drawer); data passed between screens properly; navigation is intuitive. | 0-10 | 10% |
| 6 | API Integration & Data Handling | Successfully fetches API data (fetch/axios); displays data correctly; manages loading/error states; clean async logic. | 0-15 | 15% |
| 7 | Code Quality & Documentation | Code is clean, readable, well-structured; consistent naming; meaningful comments; organized files and components. | 0-5 | 5% |
| 8 | Creativity & Innovation | Demonstrates creativity in idea, design, or extra features (animations, local storage, etc.). | 0-5 | 5% |
| 9 | Presentation & Demonstration (Optional) | Clear explanation, smooth demo, confident discussion of features and code. | 0-5 | (Bonus 5%) |
